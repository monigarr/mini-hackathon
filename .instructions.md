## 📋 Core Constraints (Non-Negotiable)

### 1. The 5-Question Budget
- **HARD LIMIT:** Agent must ask exactly ≤ 5 questions to collect all required data
- **Enforcement:** State machine counter + question validator
- **Violation:** If 6th question is triggered, state machine transitions to "complete" and does not ask
- **Implementation:** Every tool call that would prompt the user must check `question_count >= 5`
- **Example:**
  ```python
  if self.conversation_state.question_count >= 5:
      return "Thank you for the information. Let me generate your completed form..."
      # No further questions, proceed directly to tool call
  ```

### 2. Warm, Human-Like Tone
- **Mandate:** All agent responses must be conversational, reassuring, and jargon-free
- **Never:** Robotic language, interrogative tone, technical jargon
- **Always:** First-person warmth, confirmations, gentle guidance
- **Example:**
  ```
  ❌ "Please provide W-2 Box 1 value."
  ✅ "I see you have your W-2 ready. What's the total wages amount from Box 1? 
     (It's usually the largest number on the form)"
  ```
- **Standard phrases:**
  - "I'll help you file your 2025 Form 1040."
  - "Thanks for that information."
  - "Just to confirm what you told me..."
  - "Let me get your completed form ready."

### 3. No Tax Advice Beyond Data Collection
- **Guardrail:** Agent must NOT offer tax planning, optimization, or strategic advice
- **Allowed:** Data collection, form filling, basic explanations of required fields
- **Forbidden:** "You could reduce your tax bill by...", "Consider itemizing...", "You should set up an LLC..."
- **Implementation:** Prompt template includes explicit constraint:
  ```
  Do not provide tax advice, tax planning, or tax optimization strategies.
  You are collecting data to fill a form, not providing tax consulting.
  ```

### 4. Full Observability (Structured JSON Logging)
- **Requirement:** Every agent decision, tool call, and conversation turn must be logged
- **Format:** Structured JSON, one object per line (JSONL format)
- **Minimum fields:**
  ```json
  {
    "timestamp": "2026-06-24T10:00:00Z",
    "session_id": "abc123xyz",
    "event_type": "agent_response|tool_call|validation|error",
    "question_number": 1,
    "agent_message": "...",
    "tool_name": "...",
    "tool_input": {...},
    "tool_output": {...},
    "user_input": "...",
    "error_message": null
  }
  ```
- **Audit Trail:** Judge must be able to read logs and understand agent's entire decision chain
- **Implementation:** All logging goes through `src.tools.logger.log_observation()`

### 5. Deterministic W-2 Validation
- **Required boxes:** 1 (wages), 2 (federal tax withheld), 4 (social security wages), 6 (medicare wages)
- **Validation rules:**
  - All values must be positive numbers
  - Box 1 ≥ Box 2 (wages ≥ withholding)
  - Box 4 and 6 typically match Box 1 (same wages)
  - No commas or special characters in input
- **Failure handling:** Ask user to re-enter with specific guidance
- **Example:**
  ```python
  def validate_w2_box1(value: str) -> tuple[bool, str, float]:
      try:
          # Remove common formatting
          clean = value.replace(',', '').replace('$', '').strip()
          wages = float(clean)
          
          if wages < 0:
              return False, "Wages must be a positive number. Can you try again?", None
          if wages == 0:
              return False, "Wages can't be zero. Please double-check your W-2.", None
          
          return True, None, wages
      except ValueError:
          return False, "I didn't understand that number. Can you type it again?", None
  ```

### 6. 2025 Tax Table Accuracy
- **Standard Deductions (2025):**
  - Single: $14,600
  - Married Filing Jointly: $29,200
  - Dependent can claim lesser of: earned income + $450 or standard deduction
- **Tax Computation Logic:**
  - Taxable Income = Wages - Standard Deduction (minimum $0)
  - Tax = Apply 2025 tax brackets (exact rates in `src/utils/tax_tables.py`)
  - Refund/Owed = Federal Tax Withheld - Tax Liability
- **Non-Negotiable:** All tax calculations must match IRS 2025 Form 1040 instructions
- **Testing:** Every change to tax logic requires unit test verification

### 7. Form 1040 Generation Accuracy
- **Template:** `data/form_1040_template.pdf` (2025 version)
- **Fields to populate:**
  - Line 1a: Filing Status (radio button: Single/MFJ)
  - Line 1d: Total income (from W-2 Box 1)
  - Line 12: Federal income tax withheld (from W-2 Box 2)
  - Line 15: Standard deduction
  - Line 16: Taxable income (calculated)
  - Line 24: Total tax (calculated)
  - Line 33: Amount owed or refund (calculated)
- **PDF Coordinate System:** Use PyPDF2 field mapping (NOT manual coordinates)
- **Output:** PDF must be directly downloadable, not stored server-side

### State Machine (Conversation Flow)
```python
# Finite states for conversation:
# 1. START: Agent greets user, asks for W-2 data
# 2. W2_PROVIDED: W-2 data received, validate and ask for confirmation
# 3. FILING_STATUS: Ask about filing status (Single/MFJ)
# 4. DEPENDENCY: Ask if user can be claimed as dependent
# 5. CONFIRMATION: Summarize all data, ask for final confirmation
# 6. FORM_GENERATION: Call generate_1040 tool
# 7. COMPLETE: Provide download link, end conversation

# State transitions:
# START -> W2_PROVIDED (always)
# W2_PROVIDED -> FILING_STATUS (if valid) OR W2_PROVIDED (if invalid, re-ask)
# FILING_STATUS -> DEPENDENCY (always)
# DEPENDENCY -> CONFIRMATION (always)
# CONFIRMATION -> FORM_GENERATION (if user confirms)
# FORM_GENERATION -> COMPLETE (always)
```

## 🔌 Tool Implementation Pattern

All tools follow this structure:

```python
# ============================================================================
# Module: tools/example_tool.py
# Purpose: [What this tool does]
# Owner: Monica Peters
# ============================================================================

from typing import Any, Dict
import logging
from .logger import log_observation

logger = logging.getLogger(__name__)

def your_tool_name(input_param: str) -> Dict[str, Any]:
    """
    One-line description.
    
    Args:
        input_param: Description
    
    Returns:
        dict with keys: success (bool), message (str), data (Any), error (str|None)
    """
    try:
        # Validate inputs
        if not input_param:
            log_observation("validation_error", {"tool": "your_tool_name", "reason": "missing_input"})
            return {"success": False, "message": "Input required", "error": "missing_input"}
        
        # Execute tool logic
        result = perform_operation(input_param)
        
        # Log success
        log_observation("tool_success", {
            "tool": "your_tool_name",
            "input": input_param,
            "output": result
        })
        
        return {"success": True, "message": "Operation completed", "data": result, "error": None}
    
    except Exception as e:
        log_observation("tool_error", {
            "tool": "your_tool_name",
            "error": str(e),
            "traceback": traceback.format_exc()
        })
        return {"success": False, "message": "Tool failed", "error": str(e)}
```

## 📊 Conversation Flow (5-Question Budget)

**Expected conversation arc:**

| Q# | Topic | Agent asks | User provides | Constraint |
|----|-------|-----------|---------------|-----------|
| 1 | W-2 Data | "Can you share your wages (Box 1) and federal tax withheld (Box 2) from your W-2?" | Wages, withholding | Validate numbers |
| 2 | Filing Status | "Are you filing as Single or Married Filing Jointly?" | Choice | Only 2 options |
| 3 | Dependency | "Can anyone claim you as a dependent, or are you independent?" | Yes/No | Clarify if unclear |
| 4 | Confirmation | "Just to confirm: wages $40k, filing Single, independent. Correct?" | Yes/No | User validates |
| 5 | Form Gen | (Optional) "Let me generate your completed Form 1040..." | Confirmation | Direct to tool call |

**Budget management:**
- If user provides ambiguous input, do NOT count as a question—clarify within same turn
- If user asks off-topic question, do NOT count against budget—politely redirect
- After 5 questions, `state_machine.question_count >= 5` forces completion

## 🔄 Common Development Patterns

### Pattern 1: Adding a New Tool

1. Create `src/tools/new_tool.py` with standard structure
2. Add docstring with clear input/output specification
3. Implement error handling with logging
4. Write unit tests in `tests/test_new_tool.py`
5. Integrate into `src.conversation.engine.py` (LLM tool definitions)
6. Update this file with new tool documentation

### Pattern 2: Updating Prompts

1. Edit template in `src.conversation.prompts.py`
2. Ensure warm tone and constraint compliance
3. Test with real LLM call (mock if needed)
4. Verify against guardrails (no advice, no jargon)
5. Log prompt + response in observability logs

### Pattern 3: Fixing a Tax Calculation Bug

1. Reproduce in unit test
2. Fix logic in `src.utils.tax_tables.py`
3. Verify against IRS Form 1040 instructions
4. Update all affected tests
5. Generate test PDFs to verify form output
